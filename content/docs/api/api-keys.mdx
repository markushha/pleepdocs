---
title: API-ключи
description: Как создать, просмотреть и отозвать API-ключи для публичного API Pleep
---

API-ключи используются для аутентификации запросов к публичному API Pleep. Управлять ключами можно в личном кабинете в разделе **Настройки** → **API-ключи** или перейдя на страницу `/api-keys`.

## Создание API-ключа

1. Перейдите в **Настройки** → **API-ключи** в личном кабинете
2. Нажмите **Создать API-ключ**
3. Дайте ключу описательное имя (например, «Интеграция с CRM» или «Уведомления о заказах»)
4. Скопируйте ключ сразу — он отображается только один раз

<Callout title="Важно">
  Полный API-ключ отображается только в момент создания. Если вы его потеряете, необходимо отозвать старый ключ и создать новый.
</Callout>

## Просмотр API-ключей

Все активные API-ключи доступны для просмотра в личном кабинете. Каждая запись содержит имя ключа, маскированную версию ключа и дату создания.

## Отзыв API-ключа

Если API-ключ скомпрометирован или больше не нужен:

1. Перейдите в **Настройки** → **API-ключи**
2. Найдите нужный ключ
3. Нажмите **Отозвать**

Ключ отключается немедленно. Любые запросы с отозванным ключом получат ошибку `401 UNAUTHORIZED`.

## Эндпоинты управления ключами

Эти эндпоинты доступны для программного управления ключами. Они требуют JWT-аутентификации (сессию входа в личный кабинет), а не API-ключ.

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api-keys` | Создать новый API-ключ. Возвращает ключ в открытом виде один раз |
| `GET` | `/api-keys` | Список всех API-ключей вашего аккаунта |
| `DELETE` | `/api-keys/:id` | Отозвать конкретный API-ключ |

<Callout title="Примечание">
  Эндпоинты управления ключами используют сессию входа в личный кабинет (JWT), а не аутентификацию по API-ключу. Обычно ключами управляют через интерфейс личного кабинета, а не через эти эндпоинты напрямую.
</Callout>
