---
title: Үлгі хабарлама жіберу
description: Pleep API арқылы WhatsApp үлгі хабарламасын бір немесе бірнеше алушыға жіберу
---

WhatsApp үлгі хабарламасын бір немесе бірнеше телефон нөміріне жіберу. WABA Coexistence (Meta Cloud API) және ескі WABA (GupShup) арналары қолдау көрсетіледі.

## Эндпоинт

```
POST /api/v1/messages/send-template
```

## Тақырыптар

| Тақырып | Мән | Міндетті |
|---------|-----|----------|
| `X-API-Key` | Сіздің API кілтіңіз | Иә |
| `Content-Type` | `application/json` | Иә |

## Сұраным денесі

| Өріс | Тип | Міндетті | Сипаттама |
|------|-----|----------|-----------|
| `bot_id` | string | Иә | Бот UUID. Жеке кабинеттегі интеграция баптауларында табылады |
| `channel` | string | Иә | Жіберу арнасы: `waba` немесе `waba_coexistence` |
| `to` | object немесе array | Иә | Бір алушы немесе алушылар массиві |
| `template_id` | string | Иә | Үлгі идентификаторы. **Жіберулер** бөліміндегі үлгі ақпаратынан көшіріңіз |
| `template_language` | string | Жоқ | Үлгі тілінің коды. Әдепкі мәні `ru` |

### Алушы нысаны

`to` өрісіндегі әр алушы келесі құрылымға ие:

| Өріс | Тип | Міндетті | Сипаттама |
|------|-----|----------|-----------|
| `phone` | string | Иә | Халықаралық форматтағы телефон нөмірі (мысалы, `+77001234567`) |
| `params` | array | Жоқ | Осы алушы үшін үлгі айнымалы мәндері (мысалы, клиент аты) |

## Мысалдар

### Бір алушы

```json
{
  "bot_id": "your-bot-uuid",
  "channel": "waba_coexistence",
  "to": {
    "phone": "+77001234567",
    "params": ["Иван"]
  },
  "template_id": "welcome_template",
  "template_language": "ru"
}
```

### Бірнеше алушы (жаппай жіберу)

```json
{
  "bot_id": "your-bot-uuid",
  "channel": "waba",
  "to": [
    { "phone": "+77001234567", "params": ["Иван"] },
    { "phone": "+77009876543", "params": ["Мария"] },
    { "phone": "+77005551234" }
  ],
  "template_id": "welcome_template",
  "template_language": "en"
}
```

## Жауаптар

### Бір алушы — сәтті

```json
{
  "ok": true,
  "data": {
    "message_id": "wamid.HBgLNzcwMDEyMzQ1NjcVAgASGBQzRUI..."
  }
}
```

### Жаппай жіберу — сәтті

```json
{
  "ok": true,
  "data": {
    "total": 3,
    "sent": 2,
    "failed": 1,
    "results": [
      { "phone": "+77001234567", "ok": true, "message_id": "wamid.HBg..." },
      { "phone": "+77009876543", "ok": true, "message_id": "wamid.HBg..." },
      { "phone": "+77005551234", "ok": false, "error": "PROVIDER_ERROR", "message": "Жарамсыз телефон нөмірі" }
    ]
  }
}
```

## Қате кодтары

| HTTP статус | Қате коды | Сипаттама |
|-------------|-----------|-----------|
| 401 | `UNAUTHORIZED` | API кілті жоқ немесе жарамсыз |
| 403 | `FORBIDDEN` | API кілтінің бұл ботқа қатынасы жоқ |
| 404 | `BOT_NOT_FOUND` | Көрсетілген `bot_id` жоқ |
| 422 | `VALIDATION_ERROR` | Жарамсыз сұраным денесі (өрістер жоқ, қате формат) |
| 429 | `QUOTA_EXCEEDED` | Айлық хабарлама квотасы таусылды |
| 429 | `RATE_LIMITED` | Сұранымдар тым көп. Лимит — API кілтіне минутына 30 сұраным |
| 502 | `BILLING_ERROR` | Провайдер жағындағы төлем мәселесі |
| 502 | `PROVIDER_ERROR` | WhatsApp провайдерінен қате (Meta немесе GupShup) |

### Қате жауап форматы

```json
{
  "ok": false,
  "error": "VALIDATION_ERROR",
  "message": "Өріс 'template_id' міндетті"
}
```

<Callout title="Кеңес">
  `PROVIDER_ERROR` қатесін алсаңыз, үлгінің мақұлданғанын, телефон нөмірінің дұрыс екенін және WABA аккаунтыңыздың жақсы жағдайда екенін тексеріңіз.
</Callout>
