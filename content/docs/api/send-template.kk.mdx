WhatsApp үлгі хабарламасын бір немесе бірнеше телефон нөміріне жіберу. WABA Coexistence (Meta Cloud API) және ескі WABA (GupShup) арналары қолдау көрсетіледі.

## Эндпоинт

```
POST https://microservice.pleep.app/api/v1/messages/send-template
```

## Тақырыптар

| Тақырып | Мән | Міндетті |
|---------|-----|----------|
| `X-API-Key` | Сіздің API кілтіңіз | Иә |
| `Content-Type` | `application/json` | Иә |

## Бастамас бұрын

Сұраным жібермес бұрын бірнеше параметрді жинау керек: үлгі, Bot ID және арна. Төмендегі қадамдарды орындаңыз.

### 1-қадам. Үлгі жасаңыз

AI-агентіңізге өтіп, сол жақ мәзірде **Жіберулер** тармағын таңдаңыз.

![Жіберулер бөлімі](/images/api/broadcasts-section.png)

Егер үлгіңіз жоқ болса, **Жіберу жасау** батырмасын басып, үлгі жасаңыз. Үлгі мәтінінде `{{1}}`, `{{2}}` т.б. айнымалыларды қолдануға болады — мысалы, клиентке аты бойынша жүгіну үшін. Айнымалылары бар үлгілерді **тек API арқылы** жіберуге болады.

![Айнымалылары бар үлгі жасау](/images/api/create-template.png)

Жасалғаннан кейін үлгі Meta-ға тексеруге жіберіледі. **Мақұлданды** мәртебесін күтіңіз — тек содан кейін үлгіні жіберу үшін пайдалануға болады.

Үлгілер туралы толығырақ [Жіберулер → Үлгілер](/kk/broadcasts/templates) бөлімінде оқыңыз.

### 2-қадам. Template ID көшіріңіз

**Жіберулер** бөлімінде **Менің үлгілерім** блогын табыңыз. Қажет үлгіні таңдап, Template ID мәтінін басыңыз — ол автоматты түрде алмасу буферіне көшіріледі.

![Template ID көшіру](/images/api/copy-template-id.png)

### 3-қадам. Bot ID табыңыз

**Интеграциялар** бөліміне өтіп, WhatsApp Business интеграцияңызды таңдап, **Басқару** батырмасын басыңыз. Интеграция бетінде **Bot ID** өрісін көресіз — көшіру белгішесін басыңыз, Bot ID автоматты түрде көшіріледі.

![Интеграция баптауларындағы Bot ID](/images/api/bot-id.png)

### 4-қадам. Арнаны анықтаңыз

`channel` өрісінің мәні WhatsApp интеграция түріне байланысты:

## Сұраным денесі

| Өріс | Тип | Міндетті | Сипаттама |
|------|-----|----------|-----------|
| `bot_id` | string | Иә | Бот UUID. Интеграция баптауларынан көшіріңіз (3-қадам) |
| `channel` | string | Иә | Жіберу арнасы: `waba` немесе `waba_coexistence` (4-қадам) |
| `to` | object немесе array | Иә | Бір алушы немесе алушылар массиві |
| `template_id` | string | Иә | Үлгі идентификаторы. Жіберулер бөлімінен көшіріңіз (2-қадам) |
| `template_language` | string | Жоқ | Үлгі тілінің коды. Әдепкі мәні `ru` |

### Алушы нысаны

`to` өрісіндегі әр алушы келесі құрылымға ие:

| Өріс | Тип | Міндетті | Сипаттама |
|------|-----|----------|-----------|
| `phone` | string | Иә | Халықаралық форматтағы телефон нөмірі (мысалы, `+77001234567`) |
| `params` | array | Жоқ | Осы алушы үшін үлгі айнымалы мәндері (мысалы, клиент аты) |

### `params` қолдану ережелері

- `params` — **міндетті емес өріс**. Тек үлгіде айнымалылар болған кезде ғана көрсетіңіз (`{{1}}`, `{{2}}` т.б.)
- Егер үлгіде **айнымалылар жоқ** болса — `params` өрісін **мүлдем жібермеңіз**
- `params` ішіндегі бос жолдарға **тыйым салынады** — `"params": [""]` жіберсеңіз API `422 VALIDATION_ERROR` қатесін қайтарады
- Бос массив `"params": []` параметрлердің жоқтығы ретінде өңделеді

## Мысалдар

### Айнымалылары бар үлгі

```json
{
  "bot_id": "your-bot-id",
  "channel": "waba_coexistence",
  "to": {
    "phone": "+77001234567",
    "params": ["Иван", "25.03.2026"]
  },
  "template_id": "welcome_template"
}
```

### Айнымалылары жоқ үлгі

```json
{
  "bot_id": "your-bot-id",
  "channel": "waba_coexistence",
  "to": {
    "phone": "+77001234567"
  },
  "template_id": "simple_greeting"
}
```

### Бірнеше алушы (жаппай жіберу)

```json
{
  "bot_id": "your-bot-id",
  "channel": "waba_coexistence",
  "to": [
    { "phone": "+77001234567", "params": ["Иван"] },
    { "phone": "+77009876543", "params": ["Мария"] },
    { "phone": "+77005551234", "params": ["Алексей"] }
  ],
  "template_id": "welcome_template",
  "template_language": "ru"
}
```

<Callout title="Қате: params ішіндегі бос жолдар">
Келесі сұраным `422` қатесін қайтарады. Егер үлгіде айнымалылар жоқ болса, `params` өрісін жібермеңіз:

```json
{
  "to": { "phone": "+77001234567", "params": [""] }
}
```
</Callout>

## Жауаптар

### Бір алушы — сәтті

```json
{
  "ok": true,
  "data": {
    "message_id": "wamid.HBgLNzcwMDEyMzQ1NjcVAgASGBQzRUI..."
  }
}
```

### Жаппай жіберу — сәтті

```json
{
  "ok": true,
  "data": {
    "total": 3,
    "sent": 2,
    "failed": 1,
    "results": [
      { "phone": "+77001234567", "ok": true, "message_id": "wamid.HBg..." },
      { "phone": "+77009876543", "ok": true, "message_id": "wamid.HBg..." },
      { "phone": "+77005551234", "ok": false, "error": "PROVIDER_ERROR", "message": "Жарамсыз телефон нөмірі" }
    ]
  }
}
```

## Қате кодтары

| HTTP статус | Қате коды | Сипаттама |
|-------------|-----------|-----------|
| 401 | `UNAUTHORIZED` | API кілті жоқ немесе жарамсыз |
| 403 | `FORBIDDEN` | API кілтінің бұл ботқа қатынасы жоқ |
| 404 | `BOT_NOT_FOUND` | Көрсетілген `bot_id` жоқ |
| 422 | `VALIDATION_ERROR` | Жарамсыз сұраным денесі (өрістер жоқ, қате формат, params ішіндегі бос жолдар) |
| 429 | `QUOTA_EXCEEDED` | Айлық хабарлама квотасы таусылды |
| 429 | `RATE_LIMITED` | Сұранымдар тым көп. Лимит — API кілтіне минутына 30 сұраным |
| 502 | `BILLING_ERROR` | Провайдер жағындағы төлем мәселесі |
| 502 | `PROVIDER_ERROR` | WhatsApp провайдерінен қате (Meta немесе GupShup) |

### Қате жауап форматы

```json
{
  "ok": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "params contains empty strings. Either provide actual values or omit the params field entirely.",
    "status": 422
  }
}
```

### Провайдер қатесі (PROVIDER_ERROR)

`PROVIDER_ERROR` қатесі кезінде жауапта Meta немесе GupShup-тан бастапқы қате бар `provider_response` өрісі болады. Оны отладка үшін пайдаланыңыз:

```json
{
  "ok": false,
  "error": {
    "code": "PROVIDER_ERROR",
    "message": "Request failed with status code 400",
    "status": 502
  },
  "provider_response": {
    "error": {
      "message": "(#100) Invalid parameter",
      "type": "OAuthException",
      "code": 100
    }
  }
}
```

<Callout title="Кеңес">
  `PROVIDER_ERROR` алсаңыз, `provider_response` өрісін тексеріңіз — онда Meta/GupShup-тан нақты қате себебі бар. Үлгінің мақұлданғанын, телефон нөмірінің дұрыс екенін және `params` ішіндегі мәндер санының үлгідегі айнымалылар санымен сәйкес келетінін тексеріңіз.
</Callout>
