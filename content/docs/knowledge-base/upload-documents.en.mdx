Uploading documents is the fastest way to populate your AI agent's knowledge base. The system supports a wide range of formats and automatically processes content using smart processing.

## Supported Formats

- **PDF** — price lists, catalogs, presentations
- **DOCX / DOC** — Microsoft Word documents
- **XLS / XLSX** — Microsoft Excel spreadsheets
- **ODS** — OpenDocument spreadsheets
- **ODT** — OpenDocument text
- **TXT** — plain text files
- **JSON** — structured data
- **RTF / RTFD** — rich text format

## How to Upload Documents

<Steps>

### Open the Knowledge Base

Go to the **Knowledge Base** section in your agent's dashboard.

### Click "Upload"

Click the file upload button.

### Select Files

Choose one or more files from your computer. You can upload multiple files at once.

### Wait for Processing

The system will run **smart processing** — AI automatically analyzes the content, splits it into logical blocks,. The document status will change from "processing" to "indexed."

### Review the Result

After processing, you can review how the system structured the information. Edit individual blocks manually if needed.

</Steps>

## Smart processing

When you upload a document, the system does not simply extract text — it uses AI for intelligent splitting into logical elements. For example:

- A price list will be broken down into individual products with prices
- A catalog will be divided into categories and products
- Text will be split into thematic blocks

This allows the AI agent to find the right information faster and provide more accurate answers.

## Document Management

- **Status** — track the state of each file: processing, indexed, error
- **Reprocessing** — you can reindex an individual document without re-uploading it
- **Block editing** — you can create and edit individual data blocks manually

<Callout title="Google Docs and Sheets">
  Pleep supports direct import from Google Docs and Google Sheets. However, data does not update automatically — after making changes to a Google document, you need to re-import it.
</Callout>
