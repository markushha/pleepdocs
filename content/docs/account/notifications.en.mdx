title: Notifications

Configure notifications to stay informed about important events such as new customer inquiries, operator handoffs, and other activities.

## Setting up email notifications

Receive alerts via email:

1. Go to **/me** > **Notifications**.
2. Enable or disable the notification types you need.
3. Make sure the correct email address is specified.

## Notification types

You can configure notifications for the following events:

| Event | Description |
|---|---|
| New conversation | A customer started a new conversation |
| Operator handoff | The AI agent transferred a conversation to a live operator |
| Missed message | A customer wrote but did not receive a response |
| System | Platform updates, subscription changes |
| Message limit | Warning when approaching the message quota limit |

## Preference settings

For each notification type, you can choose:

- **Enabled** — notifications are sent to your email when the event occurs.
- **Disabled** — notifications are not sent.

## Team notifications

If you work as a team, each member can configure their notifications individually in the **/me** section.
