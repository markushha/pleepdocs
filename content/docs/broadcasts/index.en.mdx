Broadcasts in Pleep let you send messages and make calls to multiple customers at once. You can find the broadcast section at **Dashboard** → **Broadcast**.

## Broadcast types

Pleep offers four broadcast types, each on a separate tab:

### WhatsApp Broadcast

Send messages to multiple contacts via WhatsApp. Suitable for quick broadcasts without the need for Meta-approved templates.

### WABA Broadcast

Broadcast via the official WhatsApp Business API using **message templates** approved by Meta. This is the primary method for mass messaging — it provides high deliverability and compliance with WhatsApp policies.

### WABA Coexistence Broadcast

A hybrid mode that combines the AI agent and a live operator. The broadcast is sent via WABA, but incoming replies can be handled by either the AI agent or a human manager, depending on your settings.

### Voice Broadcast

Automated voice calls to customers via a voice AI agent. Used for calling your customer base — reminders, confirmations, informational outreach.

## Broadcast features

- **Recipient filtering** — select your audience using various criteria
- **Variable substitution** — personalize messages with the customer's name and other data
- **Scheduled sending** — schedule a broadcast for a specific time
- **Tracking and analytics** — monitor the delivery status of each message
- **Retry on failure** — automatic resend when delivery fails

<Callout title="Important">
  For WABA broadcasts, you must add a payment method in Meta Business Settings. Meta charges for each delivered template message. Learn more in [Broadcast Billing](/docs/broadcasts/payment).
</Callout>

## Getting started

1. Create a [message template](/docs/broadcasts/templates) and wait for Meta approval (for WABA broadcasts)
2. [Create a broadcast](/docs/broadcasts/creating-broadcast) — choose the type, select your audience, and send
3. Set up [billing](/docs/broadcasts/payment) if you plan to use WABA
