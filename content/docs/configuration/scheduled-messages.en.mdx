title: Scheduled Messages
description: Automated follow-up messages based on time-based triggers
---

The **Scheduled Messages** extension on the `/extensions` page lets you configure automatic message delivery to customers based on time-based triggers.

## When is this useful

- **Automatic follow-ups** — send a reminder to a customer who has not completed an action
- **Time-based triggers** — trigger a message after a specified interval following an event
- **Reminders** — automatically remind about an appointment, delivery, or upcoming event
- **Continuing a conversation** — return to a conversation after a set time

## How to configure

1. Open the **Extensions** page (`/extensions`)
2. Find the "Scheduled Messages" extension
3. Configure the trigger (how long after an event to send the message)
4. Set the message text or the rules for generating it
5. Save the settings

## Managing scheduled messages

- **View** — all scheduled messages are displayed in the extension
- **Edit** — change the text or timing before the message is sent
- **Cancel** — delete a scheduled message if it is no longer needed

## Timezone

The delivery time is based on the timezone set in agent settings (`/settings`). Make sure the timezone is configured correctly.

<Callout title="Tip">
  Schedule messages during the customer's working hours — messages received at a convenient time are more likely to get a response.
</Callout>
